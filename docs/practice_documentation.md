## 🎯 Цели и задачи практики

**Цель:** Разработка специализированного текстового редактора для создания сценариев настольных ролевых игр в рамках проекта «Подземелья Политеха».

**Задачи:**
1. Изучить workflow мастера и требования к инструментам создания сценариев
2. Реализовать систему кастомных тегов с подсветкой синтаксиса
3. Добавить поддержку базового Markdown и шаблонов сцен
4. Интегрировать вспомогательные инструменты (генератор бросков, статистика, экспорт)
5. Подготовить полный комплект отчётной документации

**Используемые технологии:**
- Язык: Python 3.8+
- GUI: tkinter (стандартная библиотека)
- Форматы: Markdown, HTML, UTF-8
- Инструменты: Git, GitHub, VS Code

## ✨ Основной функционал

- 🎨 **Подсветка синтаксиса:** Автоматическое выделение 8 категорий D&D-тегов (`[БОЙ]`, `[ПРОВЕРКА:]`, `[ЛУТ]`, `[NPC]`, `[ЛОКАЦИЯ]`, `[СОБЫТИЕ]`, `[ДИАЛОГ]`, `[СЕКРЕТ]`) уникальными цветами.
- 📝 **Поддержка Markdown:** Жирный текст через `**текст**` автоматически форматируется в интерфейсе, символы `**` затемняются для удобства чтения.
- 🧩 **Шаблоны сцен:** Быстрая вставка заготовок для локаций, NPC, боевых сцен, лута и диалогов через меню «Шаблоны».
- 🎲 **Генератор бросков:** Встроенное окно для симуляции кубиков (d4–d100) с автоматической вставкой результата в текст сценария.
- 📊 **Статистика сценария:** Модальное окно с подсчётом количества элементов каждого типа в текущем документе.
- 🌗 **Тёмная/Светлая тема:** Переключение оформления одной кнопкой или `Ctrl+T`.
- 💾 **Работа с файлами:** Создание, открытие, сохранение, экспорт в HTML с сохранением цветовой разметки и структуры.
- ⌨️ **Горячие клавиши:** Полный набор акселераторов (`Ctrl+N/O/S/E/D/T/Q`) для быстрого управления.
- 📏 **Строка состояния:** Отображение количества строк, слов и счётчиков ключевых элементов в реальном времени.

## 📅 Этапы выполнения практики

1. **Анализ требований (1–2 недели):** Изучение workflow мастера, интервью с представителями организации, формирование технического задания и списка обязательных функций.
2. **Проектирование (3 неделя):** Разработка архитектуры, обоснование выбора `tkinter` для кроссплатформенности без внешних зависимостей, прототипирование интерфейса.
3. **Разработка ядра (4–5 недели):** Реализация системы тегов, динамической подсветки синтаксиса, парсера Markdown и панели быстрых инструментов.
4. **Интеграция инструментов (6 неделя):** Добавление генератора кубиков, модуля статистики, экспорта в HTML, переключателя тем и меню шаблонов.
5. **Тестирование и отладка (7–8 недели):** Проверка на разных ОС, оптимизация производительности подсветки при большом объёме текста, исправление багов парсинга и работы с файлами.
6. **Документирование и сдача (9 неделя):** Написание отчётной документации, финальная презентация, передача материалов партнёру и преподавателю.
